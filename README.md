## I) User Stories

### 1) Admin User Stories

| **ID** | **User Stories** | **Story Points** | **Risk Factors** | **Milestone** |
|--------|-----------------|-----------------|-----------------|--------------|
| US-01 | As an Administrator, I want to assign IDs to Admins starting with #AD and #ORG to Organizers, and #AT to Attendees, so that only authorized users can access their respective functionalities based on their assigned IDs. | 8 | High | Half-way |
| US-02 | As an Administrator, I want to review society event requests so that I can approve them. | 5 | Medium | Half-way |
| US-03 | As an Administrator, I want to set up notifications for event approvals and rejections, so that societies are promptly informed of their event status. | 4 | Medium | Final |
| US-04 | As an Administrator, I want to send feedback reports to organizers when an event request is rejected, so that they understand the reasons for rejection and can make the necessary improvements before resubmitting the event. | 7 | Medium | Final |
| US-05 | As an administrator, I want organizers to gain permission to manage the public event page after an event is approved, so that they can publish event details and registration for attendees. As an Administrator, I want to restrict organizer editing access after the event deadline passes, so that they cannot make changes to past events. | 3 | Low | Final |
| US-06 | As an Administrator, I want to manage auditorium reservations, so that I can confirm venue availability before an event is officially posted. | 5 | Medium | Half-way |
| US-07 | As an Administrator, I want the system to process auditorium bookings on a First-Come-First-Served basis, so that scheduling conflicts are resolved fairly. | 5 | Medium | Final |
| US-08 | As an administrator, I want to view all approved events in a calendar so I can track upcoming campus activities. | 3 | Low | Half-way |
| US-09 | As an administrator, I want to update the event calendar when events are approved so attendees can see them. | 3 | Low | Half-way |
| US-10 | As an administrator, I want to categorize events (society-led events, school events) so users can easily browse them. | 3 | Low | Final |
| US-11 | As an administrator, I want to review and approve resource requests (like standees, certificates) so that event resources can be managed. | 3 | Low | Final |
| US-12 | As an administrator, I want to review and reject resource requests (like standees, certificates) with feedback. | 3 | Low | Final |
| US-13 | As an administrator, I want to view all event accommodation requests so that I can approve requests. | 3 | Low | Half-way |
| US-14 | As an administrator, I want to view all event accommodation requests so that I can reject relevant requests and send feedback. | 3 | Low | Final |
| US-15 | As an administrator, I want to receive accommodation details from societies so that participant data can be forwarded to the Residence office. | 5 | Medium | Half-way |
| US-16 | As an administrator, I want to download accommodation details so that I can send them to the Residence office so they can arrange rooms. | 5 | Medium | Half-way |
| US-17 | As an administrator, I want to view submitted event reports so that the CCA office can assess event performance. | 3 | Low | Final |
| US-18 | As an administrator, I want to export feedback reports so they can be reviewed by the event management office. | 5 | Medium | Final |

## III) UX Modeling

### Login Page

![Login Page](images/image1.png)

- Login page at startup to navigate to the relevant dashboard for admins, organizers and attendees by adding the required prefix for username.
  - `#AD` = Admin
  - `#ORG` = Organizer
  - `#AT` = Attendee

---

### #ORG — Organizer Dashboard

![Organizer Dashboard](images/image2.png)

- On an organizer's login, permanent sidebar view with the navigation menu.
- You get a dashboard overview with summary tabs showing total events, registrations, payments count also with any of its upcoming events.
- List of its events with the name, date, its status (Approved, Under Review, Draft, Completed) and how many registered.
- A notifications shortcut.
- Can register a new event by clicking **Register new event** or by going to **New event** from the sidebar.

**Event Proposal Form**

![Proposal Form 1](images/image3.png)
![Proposal Form 2](images/image4.png)
![Proposal Form 3](images/image5.png)

- Organizers submit events to **CCA for approval**.
- A form view consisting of 5 sections:
  - **Section 1** — Basic Event information: Event Title, Description, Event Type, Society Name, Date, Start Time, End Time, Venue and Supporting Documents.
  - **Section 2** — Guests & Participants: Expected Participants, Speakers/Guests table, VIP Protocol Toggle.
  - **Section 3** — Budget and Resources
  - **Section 4** — Event Schedule
  - **Section 5** — Accommodations
- Can be saved as draft.

**Registration Form Builder**

![Form Builder](images/image6.png)

- A form builder to create a registration form that will be required for the students to fill for registering.

**Payment Setup**

![Payment Setup](images/image7.png)

- If payment required: the amount, bank, bank account title, account number and any note.
- A preview of the ticket will be shown at the side.

**Payment Verification Dashboard**

![Payment Verification](images/image8.png)

- Shortcut metrics of the total pending, approved and rejected payments.
- Shows all participants who registered and submitted a payment proof — organizers verify and either accept or reject.

**Check-In System**

![Check-In System](images/image9.png)

- Check-in system with metrics of total registrations, checked in and remaining showing on top.
- Check-in button to click when a participant arrives.

**Equipment Requests**

![Equipment Requests](images/image10.png)

- All equipment required as summary to be submitted to CCA for booking.

**Vendor Directory**

![Vendor Directory](images/image11.png)

- All contacts of the vendors that have previously been partnered with in one place with filters to help organizers reach out easily.

**Post-Event Reports**

![Post-Event Reports](images/image12.png)

- Submitting post reports of the subsequent event to OSA.
- Shows summary on top.

**Notifications**

![Notifications](images/image13.png)

- All notifications appear here.
- Summary of total events, total registrations and pending payments.
- All notifications for approvals by CCA, payments and all equipment request confirmations. All viewed here.
